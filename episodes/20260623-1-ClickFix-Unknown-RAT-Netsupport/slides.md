## Install Zeek Extension

To query Zeek logs I use DuckDB with Yacin Nadji's `zeek` extension. First, install or update it:

```bash
duckdb -c "
FORCE INSTALL zeek FROM community;
FORCE INSTALL inet;
"
```

Every query starts with `LOAD inet;` and `LOAD zeek;`, then `read_zeek('logfile.log')` as the table.

## Detector Signatures

```
signature netsupport-cmd-encd {
    ip-proto == tcp
    payload /.*(\x0a|\x0d)CMD=ENCD(\x0a|\x0d)/
    eval NetSupport::netsupport_cmd_encd_match
}

signature netsupport-cmd-poll {
    ip-proto == tcp
    payload /.*(\x0a|\x0d)CMD=POLL(\x0a|\x0d)/

}
```

- `NetSupport::C2_Traffic_Observed_HTTP_Headers` — HTTP User-Agent or Server header contains "NetSupport"
- `NetSupport::C2_Traffic_Observed_CMD_POLL` — TCP payload contains `CMD=POLL`
- `NetSupport::C2_Traffic_Observed_CMD_ENCD` — TCP payload contains `CMD=ENCD`

> `conn.log` entries with `service=NULL` on port 443 are your canary.
